## 1. プロジェクト概要

Power BI Desktop の入門チュートリアルを提供する Web アプリ。
初学者を対象に、データ取得・整形・可視化・共有の基本フローを、ブラウザ上でステップ形式で学習できる。

---

## 2. 技術スタック

| 区分 | 採用技術 |
|------|---------|
| フレームワーク | React 18 + TypeScript |
| ビルドツール | Vite |
| スタイリング | Tailwind CSS（ダークモード: class strategy） |
| ルーティング | React Router v6（HashRouter） |
| 状態管理 | useState + カスタムフック（外部ライブラリなし） |
| 永続化 | localStorage |

---

## 3. コアデータモデル

### Step 型

```ts
type Step = {
  id: number             // 0始まりの連番
  title: string          // サイドバー・ページタイトルに表示
  estimatedMinutes: number | null  // null = 任意/参考セクション（進捗計算から除外）
  content: React.ReactNode         // JSX コンテンツ
}
```

## 4. 機能要件

### 4.1 ステップナビゲーション

- URL: `#/step/{id}` 形式のハッシュルーティング
- 不正な id は `/step/0` にリダイレクト
- ページ遷移時にスクロール位置を先頭にリセット
- キーボード操作: `←` キーで前のステップ、`→` キーで次のステップに移動
  - `<input>` / `<textarea>` フォーカス中は無効化
- 前へ/次へボタン: 各ステップ下部に表示、端のステップでは該当方向を非表示
- サイドバー: 全ステップのリストをリンク表示

### 4.2 進捗管理

- 各ステップに「完了としてマーク」ボタンを配置
- クリックで完了/未完了をトグル
- 完了状態は `localStorage` に永続化（キー: `pbi_completed_steps`、値: 完了 id の JSON 配列）
- プログレスバーに以下を表示:
  - 全体の達成率（黄色バー）
  - 現在のステップ番号 / 全ステップ数
  - 完了数 / 全ステップ数
  - 残り推定時間（`estimatedMinutes` が null でないステップの合計のうち、未通過分）

### 4.3 ダークモード

- ヘッダーにトグルボタン（☀️ ライト / 🌙 ダーク）
- 初期値の優先順位: `localStorage` > `prefers-color-scheme` > ライトモード
- 設定は `localStorage` に永続化（キー: `pbi_dark_mode`）
- `<html>` 要素の `dark` クラスを付け外しして Tailwind のダークモードを制御

### 4.4 印刷 / PDF 保存

- ヘッダーに印刷ボタン（🖨️）
- `window.print()` を呼び出す
- ヘッダー・プログレスバー・サイドバーは印刷時に非表示（`print:hidden`）

### 4.5 スクリプト表示・読み上げ機能

- 各ステップに口頭説明用スクリプトを紐付ける
  - スクリプトは `src/data/scripts.ts` で管理（`{ [stepId: number]: string }` 形式）
  - `steps.tsx` とは分離し、コンテンツ本文（学習者向け）とスクリプト（説明者向け）を独立して管理
- ヘッダーの「📜 スクリプト」ボタンでパネルの表示/非表示をトグル
  - 表示中はボタンをアンバー色でハイライト
- ブラウザ標準 Web Speech API（`SpeechSynthesis`）による読み上げ
  - 言語: `ja-JP`
  - 再生・一時停止・再開・停止の操作
  - 読み上げ速度の変更（0.75x / 1.0x / 1.25x / 1.5x / 2.0x）
  - 速度変更は再生中・一時停止中でも即時反映（現在位置から新しい速度で再スタート）
  - 外部 TTS サービスは使用しない（サーバーレス）
- カラオケ表示: `boundary` イベントで現在読み上げ中の文を黄色ハイライト
  - スクリプトを句点（。！？!?）区切りの文単位に分割し、`charIndex` で照合
  - 読み上げ中の文が自動スクロールで表示領域に入る
- ステップ遷移時に読み上げを自動停止
- `src/hooks/useSpeech.ts` カスタムフック、`src/components/ScriptPanel.tsx` コンポーネントで実装

### 4.6 用語集パネル

- ヘッダーの「📖 用語集」ボタンでパネルの表示/非表示をトグル
  - 表示中はボタンを青色でハイライト
- 右端からスライドインする固定パネル（`fixed right-0 top-22 w-80 z-40`）
  - 下端はスクリプトパネル非表示時 `bottom-0`・表示時 `bottom-44`（スクリプトパネルと重ならない）
  - ステップコンテンツを完全には隠さず、左側に内容が見える
  - 印刷時非表示（`print:hidden`）
- 用語データは `src/data/glossary.ts` で管理（`{ term: string, definition: string }[]` 形式）
- 用語名での絞り込み検索（大文字小文字を区別しない部分一致）
- `src/components/GlossaryPanel.tsx` コンポーネントで実装

### 4.7 再利用可能コンポーネント

#### Accordion

- `<details>` / `<summary>` ベース
- `defaultOpen` prop で初期展開状態を制御
- **使用場面を限定**: トラブルシューティング、FAQ (Q&A)、オプション補足情報（「後に学ぶ内容」「Copilot 参考」）、並列オプション群（保存方法・発行・共有・実践演習ステップ）のみ
- メインコンテンツのサブセクション（番号付き節 1.1〜など）には使用しない

#### Section

- 常時展開のセクション見出し + コンテンツ
- メインコンテンツのサブセクション（番号付き節、⏱️ 所要時間、🎯 対象者 など）に使用
- 折りたたみが不要な学習コンテンツに対して Accordion の代替として採用

#### CodeBlock

- 言語ラベル（省略可）+ コード表示 + コピーボタン
- コピー後 2 秒間「✓ コピー済」表示

### Sidebar

- 幅: `w-56`（224px）、スクロール追従（sticky top-88px）
- 各ステップのリンクに状態アイコンを表示:
  - 完了: `✅`
  - 現在: `▶`（黄色ハイライト + 右ボーダー）
  - その他: `□`

### ScriptPanel（フッター固定）

- `fixed bottom-0 left-0 right-0 z-40`、印刷時非表示（`print:hidden`）
- ヘッダーの「📜 スクリプト」ボタンでトグル（Layout が `showScript` 状態を管理）
- コントロール行: 再生/一時停止/再開/停止ボタン + 速度選択（0.75x〜2.0x）+ 閉じるボタン
- スクリプトテキスト: `max-h-20 overflow-y-auto`、コンテナ内スクロールのみ（ページスクロールに干渉しない）
- カラオケハイライト: `boundary` イベントの `charIndex` で現在文を特定し黄色ハイライト

## 7. コンテンツ管理ルール

- **`CONTENTS.md` がコンテンツの唯一の骨子**
- 変更フロー: `CONTENTS.md` → `src/data/steps.tsx` → 必要に応じてコンポーネント
- 両ファイルは常にセットで更新する（乖離禁止）
- React アプリ側では Markdown の制約を超えた視覚表現（フロー図・比較カード・スタイル付きテーブル等）を積極的に実装する

## 9. 計画中の機能追加

### 9.2 実測タイマーと進捗ペース確認

**背景**: チュートリアルの目標所要時間は約1時間。実際にかかった時間を計測し、
ペースが適切かをリアルタイムで確認できる UI を提供する。

**要件:**

- セッション開始時刻を記録し、経過時間をリアルタイム表示
  - 「開始」ボタンで計測スタート（手動開始）
  - 経過時間と目標時間（推定分の合計）を比較してペインを表示
- 現在ステップの「推定ペース」との差分表示
  - 例: 現在ステップまでの推定累積時間 vs. 実際の経過時間
  - 遅れている / 順調 / 早い の3段階で視覚的にフィードバック
- 計測データは `localStorage` に保存し、ページリロード後も継続できる

### 9.4 ステップ内ミニ目次

**背景**: 手順数の多い長いステップ（例: ステップ6 ピボット解除）で「どこまで進んだか」を把握しにくい。

**要件:**

- `estimatedMinutes` が一定以上の長いステップでのみ表示（短いステップには不要）
- ステップコンテンツ内の見出し（`<h2>`, `<h3>` 相当）を自動収集してリスト化
- クリックで該当箇所にスムーズスクロール
- コンテンツ側に特別なマークアップは不要（DOM を走査して自動生成）

### 9.5 ステップごとのメモ機能

**背景**: 「ここで詰まった」「自分のデータではこう対応する」など、学習者個人の気づきをその場で記録できる手段を提供する。

**要件:**

- 各ステップのコンテンツ末尾（完了ボタンの前）にメモ入力欄を配置
- 入力内容は `localStorage` に保存（キー: `pbi_notes`、値: `{ [stepId]: string }` の JSON）
- メモが入力済みのステップはサイドバーのアイコンで区別できる（任意）

### 9.6 チェックポイントクイズ

**背景**: 読んだだけで理解した気になるのを防ぎ、自習での理解度確認を促す。

**要件:**

- 主要ステップの末尾に選択式クイズを配置（全ステップに必須ではない）
- クイズデータは `src/data/quizzes.ts` で管理し、`steps.tsx` とは分離
  - `{ stepId: number, question: string, options: string[], correctIndex: number, explanation: string }[]` 形式
- 正解・不正解をその場でフィードバック表示
- 不正解時は該当セクションへのリンクを表示
- 回答状態は保存しない（何度でも回答可）

### 9.7 完了履歴タイムライン

**背景**: 複数回に分けて学習する場合に「前回どこまでやったか」「どのステップに時間がかかったか」を可視化する。

**要件:**

- 各ステップの完了ボタンを押した日時を記録
- 専用の履歴画面（またはサイドパネル）でタイムライン表示
- 完了日時は `localStorage` に保存（キー: `pbi_completion_timestamps`、値: `{ [stepId]: ISO8601文字列 }` の JSON）
- 既存の `pbi_completed_steps` との整合性を保つ（完了取り消し時は日時も削除）

## 10. 既知の制約・未実装事項

- モバイル対応なし（サイドバーが常時表示のため小画面で窮屈）
- 検索機能なし
- ステップ間のデータ連携なし（各ステップは独立）
- 多言語対応なし（日本語専用）
