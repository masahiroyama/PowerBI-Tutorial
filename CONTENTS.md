## 1. データ解析の基礎

### 1.1 解析しやすいデータ vs. 人が見やすいデータ

多くの人は、Excel スプレッドシートを見やすくするために、表を工夫して作成します。
しかし、**コンピュータがデータを分析しやすい形**と**人間が見やすい形**は異なります。

#### ❌ 人間には見やすいが、解析に不適切な形（雑然データ）

```csv
タスク,1月,2月,3月,4月,5月
営業報告,10,12,11,13,14
データ分析,8,8,9,7,8
マーケティング,15,14,16,15,17
プロジェクト管理,12,11,12,13,12
システム開発,20,22,21,19,20
```

**この形式の問題点：**
- 全期間のデータを集計するのに、複数の列を参照する必要がある
- グラフやピボットテーブルで分析するには、前処理が必要

#### ✅ コンピュータが処理しやすい形（整然データ）

```csv
タスク,月,工数
営業報告,1月,10
営業報告,2月,12
営業報告,3月,11
営業報告,4月,13
営業報告,5月,14
データ分析,1月,8
データ分析,2月,8
データ分析,3月,9
データ分析,4月,7
データ分析,5月,8
...
```

**この形式の利点：**
- 各行が1つの観測値（1つのデータ）を表す
- 列の追加が容易（新しい属性があれば列を追加するだけ）
- 集計や集団ごとの分析が単純
- ピボットテーブルやグラフに直接使用可能
- データベースに格納しやすい
- プログラミングやBIツールでの処理が効率的

#### ❌ 雑然データ（よくある別パターン）

「見やすく整えた」つもりが解析できないデータになる例：

| 部署 | 担当者 | 売上（万円） | ステータス |
| ---- | ------ | ------------ | ---------- |
| 営業部（3行結合）① | 田中 | 120（先月比+8）③ | 進行中 |
| （結合）① | 鈴木 | 85（先月比−5）③ | ↑ ② |
| （結合）① | 山田 | 200（先月比+15）③ | 完了 |

- **① セルの結合**：「営業部」が3行にまたがるため、各行に部署情報が入らず、フィルタや集計ができない
- **② 「↑」による省略**：目で見れば伝わるが、プログラムは前の行の値を引き継がないため「↑」という文字列として読み込まれる
- **③ セル内の複数情報**：「120（先月比+8）」のように数値と注記が混在すると、Power BI などのツールは数値と注記を別々に読み取ることができず、集計やグラフ化でエラーが起きやすい。AI（Copilot・ChatGPT など）はある程度読み取れるが、大量データでは見落としや誤解釈が積み上がるリスクがある

#### ✅ 整然データに直すとどうなる？（折り畳み・受講者が考えてから確認）

①②③ をすべて解消した整然データ版：

| 部署 | 担当者 | 売上（万円） | 前月比（万円） | ステータス |
| ---- | ------ | ------------ | -------------- | ---------- |
| 営業部 | 田中 | 120 | +8 | 進行中 |
| 営業部 | 鈴木 | 85 | −5 | 進行中 |
| 営業部 | 山田 | 200 | +15 | 完了 |

- ① 結合を解除：各行に「営業部」を明示し、フィルタ・集計が可能になった
- ② 「↑」を実値に置換：鈴木行のステータスを「進行中」と明示した
- ③ 列を分離：「120（先月比+8）」を「売上」と「前月比」の2列に分け、各セルが1値になった

#### 💡 雑然データが「問題になるとき・ならないとき」

判断軸は**更新頻度ではなく、解析ツールに通すかどうか**です。

- **問題にならないケース**：人に見せるだけで解析ツールには通さない／印刷・配布・プレゼン用の固定レポート
- **問題になるケース**：Power BI・AI などで分析したい／集計・フィルタ・グラフ化したい

**実務上のポイント：** 「今は配布するだけ」でも、将来的に分析したくなる可能性があるなら、「整然データに変換しやすい状態」を保っておくことが重要です。セル結合しない・↑省略しない・1セル1値、の3点を守るだけで、後から Power BI や AI に通せる状態になります。

### 1.2 データ解析とは？
データ解析とは、大量のデータから意味のある情報やパターンを抽出し、ビジネスの意思決定を支援するプロセスです。
「人が見やすい形（雑然データ）」を「コンピュータが処理しやすい形（整然データ）」に変換することが、その第一歩となります。

```
生データ（雑然データ）
    ↓ データ整形・加工
整理されたデータ（整然データ）
    ↓ 分析・可視化
洞察・ビジネス上の価値
```

### 1.3 なぜ整然データが重要か？

- **Power BI、Tableau などの BI ツールは整然データを想定**して設計されている
- **AI との相性が良い**  
  整然データ、もしくは簡単に整然データに変換できる形にしておくと、AI がデータの構造を推測する手間が減り、より的確な分析結果が得られやすくなります。
- **データの保守・更新が簡単**
- **複数のデータセットの統合が容易**

### 1.4 このチュートリアルでの実践

このチュートリアルで使用するサンプルデータは、意図的に**雑然データ（ピボット形式）**として提供しています。
Power Query でピボット解除を行い、**整然データに変換するプロセス**を学ぶことで、実務的なデータ整形スキルを習得できます。

## 2. Power BI とは？
Power BI は、バラバラな場所にあるデータを接続し、視覚化し、組織内で共有するためのビジネスインテリジェンス（BI）ツールです。

- **Power BI Desktop**: レポートを作成するための無料デスクトップアプリ。
- **Power BI サービス**: 作成したレポートをオンラインで共有・閲覧するためのクラウドサービス。

## 3. Excelのテーブル機能を使おう

多くの人がExcelでデータを作成するとき、通常の範囲としてデータを入力しています。
しかし、**Excelのテーブル機能**を使うことで、データ管理とPower BIとの連携がはるかに効率的になります。

### 3.1 テーブル機能とは

テーブル機能は、Excelのデータを**構造化された形式**に変換する機能です。
テーブルにすることで、多くの利点が得られます。

### 3.2 テーブル機能の利点

#### 📋 列名での式作成
通常の参照：
```excel
=SUM(B2:B100)
```

テーブルの式：
```excel
=SUM(Sales[Amount])
      ↑       ↑
  テーブル名  列名（見出し行の名前）
（テーブルに付けた名前）
```

**メリット：**
- 式が読みやすく、意図が明確
- 新しい行を追加すると、自動的に計算対象に含まれる
- 行番号を気にしなくてよい

#### 🔄 Power BIとの連携が簡単
- ヘッダー行を自動認識
- 表の終わりを自動判定
- データ構造の変更に対応しやすい

#### ✅ データの一貫性が保たれる
- 列名が固定されるため、誤操作による列の削除を防止
- 自動的に行のスタイルが統一される

### 3.3 テーブルの作成方法

1. **データを選択**
   - 見出し行を含めて、データ範囲全体を選択
   - ショートカット：先頭セル（A1）をクリック → Ctrl+Shift+End で末尾まで一括選択

2. **テーブルに変換**
   - **挿入** タブ → **テーブル** をクリック
   - または **データ** タブ → **テーブルとして書式設定**
   - 「先頭行をテーブルの見出しとして使用する」にチェックが入っていることを確認して **OK**

3. **テーブル名の設定**
   - テーブルを作成すると、Excelのリボン（画面上部のメニューバー）に「**テーブルデザイン**」タブが新たに表示される
   - 「テーブルデザイン」タブをクリック（テーブルのセルをクリックしている状態でのみ表示される）
   - リボンの**左端**に「テーブル名：」という入力欄が表示される（初期値は「テーブル1」のような自動名称）
   - 入力欄の内容を消して、分かりやすい名前（例：`Sales`、`ProjectTasks`）を入力してEnterキーで確定
   - ⚠️ テーブル外のセルをクリックすると「テーブルデザイン」タブは消える（テーブル内のセルをクリックすると再表示される）

### 3.4 テーブルでのデータ追加

テーブルの最後の行の下に新しいデータを入力すると、自動的にテーブルが拡張されます。
新しい行を追加するたびに、以下が自動的に行われます：
- スタイルが適用される
- テーブル内の式が自動的に新しい行に拡張される
- Power BIが最新のデータを認識する

### 3.5 テーブルはいつ使うべき？

**テーブルを使うべき場合：**
- ✅ 定期的に更新されるデータ
- ✅ Power BI や他のBIツールに連携するデータ
- ✅ 複数人で共有するデータ
- ✅ 計算式を含むデータ

**テーブルが不要な場合：**
- 一時的なメモや計算用の表
- 形式が固定されていない試行錯誤用

### 3.6 Power BIでの使用方法

**このチュートリアルでの活用：**
このチュートリアルで使用するサンプルデータは、意図的に**ピボット形式（雑然データ）**として提供しています。

セクション6の「ステップ2：データの整形 - ピボット解除」で、以下を実践します：
1. Excelテーブル形式で管理されたデータ（またはCSV）
2. Power Query でピボット解除し、**整然データに変換**
3. 変換されたデータを可視化

このプロセスを習得することで、実務でも効率的にデータ分析ができるようになります。

### 3.7 サンプルデータをテーブルに変換する

ダウンロードした `sample-data.xlsx` を開き、Power BI で使用するためにテーブルに変換して保存します。

1. データ範囲（A1〜AL31）を選択（A1 をクリック → Ctrl+Shift+End で一括選択）
2. **挿入** タブ → **テーブル** をクリック
3. 「先頭行をテーブルの見出しとして使用する」にチェックが入っていることを確認し **OK**（範囲は `=$A$1:$AL$31`）
4. 「**テーブルデザイン**」タブをクリックし、リボン**左端**の「テーブル名：」入力欄を `ProjectTasks` に書き換えてEnterを押す
5. **ファイルを保存**（ファイル名：`project_tasks.xlsx`、ファイルの種類：Excel ブック（.xlsx）、保存場所：デスクトップ推奨）

以上で、Power BI への取り込み準備が完了です。

## 4. 基本的なワークフロー
Power BI での作業は、大きく分けて以下の5つのステップで進みます。

1. **データの取得 (Get Data)**: 様々なソースからデータを読み込む。
2. **データの整形 (Power Query)**: データの汚れを落とし、使いやすく加工する。
3. **データモデリング**: テーブル同士の関連付けを行う。
4. **視覚化 (Report)**: グラフやチャートを配置する。
5. **発行 (Publish)**: クラウドにアップロードして共有する。

## 6. ステップ 2: データの整形 - ピボット解除（15分）

### 6.1 データ構造について

準備したサンプルデータ `project_tasks.xlsx`（テーブル名：`ProjectTasks`）は、以下の**ピボット形式**（日付が列として横に並ぶ形式。雑然データとも呼ぶ）です：

| プロジェクト | タスク | 2022/01/01 | 2022/02/01 | … | 2024/12/01 |
| ------------ | ------ | ---------- | ---------- | - | ---------- |
| プロジェクトA | 要件定義 | 14 | 8 | … | 11 |
| プロジェクトA | 設計 | 7 | 19 | … | 6 |
| … | … | … | … | … | … |

このデータは**人間には見やすい形式**ですが、Power BI での分析には不適切です。
セクション1で学んだ「整然データ」に変換する必要があります。

### 6.2 Power Query エディタの画面構成

ステップ5で「データの変換」をクリックすると Power Query エディタが開きます。画面は以下の3エリアで構成されています：

- **左パネル**：クエリ一覧
- **中央**：データプレビュー（現在のデータが表示される）
- **右パネル**：「適用したステップ」（変換の履歴）

### 6.3 データ構造の確認
- 現在のデータは「ワイド形式」（「プロジェクト」「タスク」の固定列 + 日付が列として横に並ぶ形式）です
- 分析に適した「ロング形式」（日付を行として展開）に変換する必要があります

### 6.4 ピボット解除の実行

**推奨手順（「他の列のピボット解除」を使う方法）：**

1. 「プロジェクト」列のヘッダーをクリック、続けて **Ctrl キーを押しながら「タスク」列のヘッダーをクリック** して2列を選択
2. いずれかの列ヘッダーを **右クリック** → **「他の列のピボット解除」** を選択

この方法は、将来さらに月の列が追加されても自動的に処理されるため実務向きです。

**別の方法（列を手動で選択する方法）：**

1. 「2022/01/01」列ヘッダーをクリック → Shift キーを押しながら「2024/12/01」をクリックして全日付列を選択
2. **「変換」** タブ → **「ピボットの解除」** をクリック

### 6.5 列のリネーム
ピボット解除後、「属性」と「値」という列が自動生成されます。これは、ピボット解除によって「元の列名（2022/01/01 など）」が「属性」列に、「元のセルの数値」が「値」列に格納されるためです。
- 「属性」を右クリック → **「名前変更」** → 「月」
- 「値」を右クリック → **「名前変更」** → 「工数」

### 6.6 データ型の確認
- 「月」列を選択し、データ型を **「日付」** に設定（日付型にすることで、Power BI の時系列グラフで日付順に正しく並びます）
- 「工数」列を選択し、データ型を **「整数」** に設定（工数は小数にならないため整数型を指定します）
- **「閉じて適用」** をクリックして Power Query を閉じ、データを Power BI に取り込みます

最終的なデータ構造：
```
プロジェクト | タスク | 月 | 工数
プロジェクトA | 要件定義 | 2022/01/01 | 14
プロジェクトA | 要件定義 | 2022/02/01 | 8
...
```

## 7. ステップ 3: データモデリング（5分）

### 7.1 単一テーブルの確認

このチュートリアルで使用するサンプルデータは、**単一のテーブル**で構成されています。

**データモデリングとは：**
複数のテーブルを組み合わせて、分析しやすい構造を作ることです。

**このチュートリアルでは不要な理由：**
- 単一テーブルのため、リレーションシップ（関連付け）は必要ない
- 複数テーブルの関連付けは、より高度なトピックなので、基礎習得後に学ぶ

### 7.2 後に学ぶ内容（参考）

実務では、通常、複数のテーブルを組み合わせて分析を行います：

## 8. ステップ 4: 視覚化 - レポートの作成（20分）

### 8.0 完成イメージ

このセクションで作成するレポートのレイアウトは以下のとおりです：

   - **X軸** に「月」をドラッグ

### 📚 整然データの3つの原則（Tidy Data）

データ科学者 Hadley Wickham が提唱した概念です。実務では直感的に理解できれば十分ですが、より深く理解したい方はこちらを参照してください。

1. **各行は1つの観測値である**：同一の対象（「タスク×月」の組み合わせ）に関するデータが1行に集約される
2. **各列は1つの変数である**：列ごとに1つの属性のみを持つ
3. **各セルは1つの値である**：セルには単一の値のみを含む

### 🤖 AI を活用した学習

Power BI の操作で詰まったとき、チャットベースのAI（Copilot、Claude、ChatGPT など）をうまく活用すると的確なアドバイスをすぐに得られます。

#### コツ：スクリーンショット＋やりたいこと

**詰まった画面のスクリーンショット**と**やりたいこと**をセットでAIに伝えるのが効果的です。
画面の状況を把握したAIが、抽象的なアドバイスではなく具体的な操作手順を教えてくれます。

活用場面の例：
- Power Query でデータ変換の手順がわからないとき
- DAX の計算式の書き方がわからないとき
- グラフの書式設定でどのオプションを使えばいいかわからないとき
- エラーメッセージの意味がわからないとき

質問の例：
> 「このスクリーンショットは Power BI の Power Query エディタです。月の列を昇順に並べ替えたいのですが、どうすればいいですか？」

## 11. よくある質問（FAQ）

**Q: Power Query でエラーが出てピボット解除できない**
- A: データ型を確認してください。「月」列が数値になっていないか確認し、必要に応じて日付型に設定してください。

**Q: 月の順序が1月から5月ではなく、ランダムになっている**
- A: 「月」列をソートします。Power Query で「昇順」を選択し、カスタムソートで月の順序を指定できます。

**Q: レポートが重くなってきた**
- A: 不要な列を削除してください。Power Query で不要な列を削除することで、ファイルサイズが小さくなります。

**Q: 複数のユーザーが同時に編集したい**
- A: Power BI Service の「編集」機能を使用すると、複数ユーザーによる同時編集が可能です。
