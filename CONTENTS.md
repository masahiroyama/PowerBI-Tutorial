## 1. データ解析とは？

### 1.1 データ解析の定義
データ解析とは、大量のデータから意味のある情報やパターンを抽出し、ビジネスの意思決定を支援するプロセスです。

```
生データ（雑然データ）
    ↓ データ整形・加工
整理されたデータ（整然データ）
    ↓ 分析・可視化
洞察・ビジネス上の価値
```

### 1.2 データ解析しやすいデータ vs. 人が見やすいデータ

多くの人は、Excel スプレッドシートを見やすくするために、表を工夫して作成します。
しかし、**コンピュータがデータを分析しやすい形**と**人間が見やすい形**は異なります。

#### ❌ 人間には見やすいが、解析に不適切な形（雑然データ）

```csv
タスク,1月,2月,3月,4月,5月
営業報告,10,12,11,13,14
データ分析,8,8,9,7,8
マーケティング,15,14,16,15,17
プロジェクト管理,12,11,12,13,12
システム開発,20,22,21,19,20
```

**この形式の問題点：**
- 月が列になっているため、新しい月を追加するたびに列を増やす必要がある
- 全期間のデータを集計するのに、複数の列を参照する必要がある
- 同じ構造のデータを追加すると、管理が複雑になる
- グラフやピボットテーブルで分析するには、前処理が必要

#### ✅ コンピュータが処理しやすい形（整然データ）

```csv
タスク,月,工数
営業報告,1月,10
営業報告,2月,12
営業報告,3月,11
営業報告,4月,13
営業報告,5月,14
データ分析,1月,8
データ分析,2月,8
データ分析,3月,9
データ分析,4月,7
データ分析,5月,8
...
```

**この形式の利点：**
- 各行が1つの観測値（1つのデータ）を表す
- 列の追加が容易（新しい属性があれば列を追加するだけ）
- 集計や集団ごとの分析が単純
- ピボットテーブルやグラフに直接使用可能
- データベースに格納しやすい
- プログラミングやBIツールでの処理が効率的

### 1.3 整然データ（Tidy Data）の3つの原則

これはデータ科学者 Hadley Wickham が提唱した概念です：

1. **各行は1つの観測値である**
   - 同一の対象（この例では「タスク×月」の組み合わせ）に関するデータが1行に集約される

2. **各列は1つの変数である**
   - タスク列には「タスク」、月列には「月」というように、列ごとに1つの属性のみを持つ

3. **各セルは1つの値である**
   - セルには複数の値が混在せず、単一の値のみを含む

### 1.4 なぜ整然データが重要か？

- **Power BI、Tableau などの BI ツールは整然データを想定**して設計されている
- **SQL や Python などのプログラミング言語との相性が良い**
- **機械学習モデルの学習に必要**
- **データの保守・更新が簡単**
- **複数のデータセットの統合が容易**

### 1.5 このチュートリアルでの実践

このチュートリアルで使用するサンプルデータは、意図的に**雑然データ（ピボット形式）**として提供しています。
Power Query でピボット解除を行い、**整然データに変換するプロセス**を学ぶことで、実務的なデータ整形スキルを習得できます。

## 2. Power BI とは？
Power BI は、バラバラな場所にあるデータを接続し、視覚化し、組織内で共有するためのビジネスインテリジェンス（BI）ツールです。

- **Power BI Desktop**: レポートを作成するための無料デスクトップアプリ。
- **Power BI サービス**: 作成したレポートをオンラインで共有・閲覧するためのクラウドサービス。

## 3. Excelのテーブル機能を使おう

多くの人がExcelでデータを作成するとき、通常の範囲としてデータを入力しています。
しかし、**Excelのテーブル機能**を使うことで、データ管理とPower BIとの連携がはるかに効率的になります。

### 3.1 テーブル機能とは

テーブル機能は、Excelのデータを**構造化された形式**に変換する機能です。
テーブルにすることで、多くの利点が得られます。

### 3.2 テーブル機能の利点

#### 📋 列名での式作成
通常の参照：
```excel
=SUM(B2:B100)
```

テーブルの式：
```excel
=SUM(Sales[Amount])  ← 列名「Amount」を直接参照
```

**メリット：**
- 式が読みやすく、意図が明確
- 新しい行を追加すると、自動的に計算対象に含まれる
- 行番号を気にしなくてよい

#### 🔄 Power BIとの連携が簡単
- ヘッダー行を自動認識
- 表の終わりを自動判定
- データ構造の変更に対応しやすい

#### ✅ データの一貫性が保たれる
- 列名が固定されるため、誤操作による列の削除を防止
- 自動的に行のスタイルが統一される

### 3.3 テーブルの作成方法

1. **データを選択**
   - 見出し行を含めて、データ範囲全体を選択

2. **テーブルに変換**
   - **挿入** タブ → **テーブル** をクリック
   - または **データ** タブ → **テーブルとして書式設定**

3. **テーブル名の設定**
   - テーブルを作成すると、Excelのリボン（画面上部のメニューバー）に「**テーブルデザイン**」タブが新たに表示される
   - 「テーブルデザイン」タブをクリック（テーブルのセルをクリックしている状態でのみ表示される）
   - リボンの**左端**に「テーブル名：」という入力欄が表示される（初期値は「テーブル1」のような自動名称）
   - 入力欄の内容を消して、分かりやすい名前（例：`Sales`、`ProjectTasks`）を入力してEnterキーで確定
   - ⚠️ テーブル外のセルをクリックすると「テーブルデザイン」タブは消える（テーブル内のセルをクリックすると再表示される）

### 3.4 テーブルでのデータ追加

テーブルの最後の行の下に新しいデータを入力すると、自動的にテーブルが拡張されます。
新しい行を追加するたびに、以下が自動的に行われます：
- スタイルが適用される
- テーブル内の式が自動的に新しい行に拡張される
- Power BIが最新のデータを認識する

### 3.5 テーブルはいつ使うべき？

**テーブルを使うべき場合：**
- ✅ 定期的に更新されるデータ
- ✅ Power BI や他のBIツールに連携するデータ
- ✅ 複数人で共有するデータ
- ✅ 計算式を含むデータ

**テーブルが不要な場合：**
- 一時的なメモや計算用の表
- 形式が固定されていない試行錯誤用

### 3.6 Power BIでの使用方法

**このチュートリアルでの活用：**
このチュートリアルで使用するサンプルデータは、意図的に**ピボット形式（雑然データ）**として提供しています。

セクション6の「ステップ2：データの整形 - ピボット解除」で、以下を実践します：
1. Excelテーブル形式で管理されたデータ（またはCSV）
2. Power Query でピボット解除し、**整然データに変換**
3. 変換されたデータを可視化

このプロセスを習得することで、実務でも効率的にデータ分析ができるようになります。

## 4. 基本的なワークフロー
Power BI での作業は、大きく分けて以下の5つのステップで進みます。

1. **データの取得 (Get Data)**: 様々なソースからデータを読み込む。
2. **データの整形 (Power Query)**: データの汚れを落とし、使いやすく加工する。
3. **データモデリング**: テーブル同士の関連付けを行う。
4. **視覚化 (Report)**: グラフやチャートを配置する。
5. **発行 (Publish)**: クラウドにアップロードして共有する。

## 6. ステップ 2: データの整形 - ピボット解除（15分）

### 6.1 データ構造について

準備したサンプルデータ `project_tasks.xlsx`（テーブル名：`ProjectTasks`）は、以下のピボット形式（雑然データ）です：

| タスク | 1月 | 2月 | 3月 | 4月 | 5月 |
| ------ | --- | --- | --- | --- | --- |
| 営業報告 | 10 | 12 | 11 | 13 | 14 |
| ... | | | | | |

このデータは**人間には見やすい形式**ですが、Power BI での分析には不適切です。
セクション1で学んだ「整然データ」に変換する必要があります。

### 6.2 Power Query エディタを開く
- 読み込み後、自動的に **Power Query エディタ** が開きます。
- もし開かない場合は、**「クエリを編集」** をクリックしてください。

### 6.3 データ構造の確認
- 現在のデータは「ワイド形式」（列に月が並んでいる）です
- 分析に適した「ロング形式」（月を行として展開）に変換する必要があります

### 6.4 ピボット解除の実行

**推奨手順（「他の列のピボット解除」を使う方法）：**

1. 「タスク」列のヘッダーを **右クリック**
2. **「他の列のピボット解除」** を選択

この方法は、将来6月・7月などの列が追加されても自動的に処理されるため実務向きです。

**別の方法（列を手動で選択する方法）：**

1. 「1月」の列ヘッダーをクリック → Shift キーを押しながら「5月」をクリックして全月列を選択
2. **「変換」** タブ → **「ピボットの解除」** をクリック

### 6.5 列のリネーム
ピボット解除後、「属性」と「値」という列が自動生成されます。
- 「属性」を右クリック → **「名前変更」** → 「月」
- 「値」を右クリック → **「名前変更」** → 「工数」

### 6.6 データ型の確認
- 「工数」列を選択し、データ型を **「整数」** に設定
- **「完了」** をクリック

最終的なデータ構造：
```
タスク | 月 | 工数
営業報告 | 1月 | 10
営業報告 | 2月 | 12
営業報告 | 3月 | 11
...
```

## 8. ステップ 4: 視覚化 - レポートの作成（20分）

### 8.0 完成イメージ

このセクションで作成するレポートのレイアウトは以下のとおりです：

- **上部**：スライサー（タスク名でフィルタリング）
- **左中段**：折れ線グラフ（月別工数推移、タスク別色分け）
- **右中段**：テーブル（タスク・月・工数の詳細データ）

スライサーを操作すると、グラフとテーブルが連動して絞り込まれます。

### 8.1 レポートビューに切り替え
- 左側の「レポート」アイコンをクリック

### 8.2 グラフの作成：タスク別の工数を比較する折れ線グラフ

1. キャンバスの空白部分をクリック
2. 右側の **「視覚化」ペイン** から **「折れ線グラフ」** アイコンをクリック
3. 右側の **「フィールド」ペイン** から：
   - **X軸** に「月」をドラッグ
   - **Y軸** に「工数」をドラッグ（自動集計は合計）
   - **凡例** に「タスク」をドラッグ

### 8.3 グラフの書式設定
1. グラフをクリック
2. **「書式」** タブで：
   - タイトルを追加：「タスク別の月ごとの工数推移」
   - 凡例を右側に配置
   - グリッドラインを有効化

### 8.4 テーブルの作成：詳細データの表示
1. 別の場所に新しいビジュアルを追加
2. **「テーブル」** を選択
3. フィールドから「タスク」「月」「工数」をすべてドラッグ
4. テーブルをリサイズして整理

### 8.5 スライサーの追加：インタラクティブなフィルタリング
1. 上部に新しいビジュアルを追加
2. **「スライサー」** を選択
3. 「タスク」をドラッグ
4. グラフとテーブルが自動的にフィルタリングされることを確認

4. 同僚に共有

### 🤖 AI を活用した学習

Power BI の操作で詰まったとき、チャットベースのAI（Copilot、Claude、ChatGPT など）をうまく活用すると的確なアドバイスをすぐに得られます。

#### コツ：スクリーンショット＋やりたいこと

**詰まった画面のスクリーンショット**と**やりたいこと**をセットでAIに伝えるのが効果的です。
画面の状況を把握したAIが、抽象的なアドバイスではなく具体的な操作手順を教えてくれます。

活用場面の例：
- Power Query でデータ変換の手順がわからないとき
- DAX の計算式の書き方がわからないとき
- グラフの書式設定でどのオプションを使えばいいかわからないとき
- エラーメッセージの意味がわからないとき

質問の例：
> 「このスクリーンショットは Power BI の Power Query エディタです。月の列を昇順に並べ替えたいのですが、どうすればいいですか？」

#### Power BI 組み込みの Copilot（参考）

Power BI 自体にもCopilot機能が組み込まれており、自然言語でレポートの作成や質問ができます。
ただし、利用には **Microsoft Fabric または Power BI Premium ライセンス** が必要です。

## 10. よくある質問（FAQ）

**Q: Power Query でエラーが出てピボット解除できない**
- A: データ型を確認してください。「月」が数値になっていないか確認し、必要に応じてテキスト型に設定してください。

**Q: 月の順序が1月から5月ではなく、ランダムになっている**
- A: 「月」列をソートします。Power Query で「昇順」を選択し、カスタムソートで月の順序を指定できます。

**Q: レポートが重くなってきた**
- A: 不要な列を削除してください。Power Query で不要な列を削除することで、ファイルサイズが小さくなります。

**Q: 複数のユーザーが同時に編集したい**
- A: Power BI Service の「編集」機能を使用すると、複数ユーザーによる同時編集が可能です。
